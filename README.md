# xing-dst-folkmod

朋友联机用的《饥荒联机版》轻量优化 Mod。

当前版本以“尽量简单、尽量稳”为目标，优先复用原版容器、堆叠、地图和交易机制，避免大规模重写原版逻辑。

## 功能简单介绍

1. 普通箱子固定扩容到 300 格，冰箱固定扩容到 100 格。
2. 所有可堆叠物品的堆叠上限统一提升到 300。
3. 新增一个黑色垃圾箱，容量固定 100 格，物品放进去 2 天后会变成灰烬。
4. 切斯特和哈奇固定扩容到 300 格，并且无敌、不会吸引怪物攻击。
5. 队友之间会共享已经探索过的地图视野，但不会直接开启全图。
6. 草席卷支持在白天正常使用。
7. 冰箱和盐盒的保鲜效果增强，食物保鲜时间提升到原来的 2 倍。

## 当前目录

- `modinfo.lua`: 模组信息
- `modmain.lua`: 主要功能实现
- `prefabs/folkmod_trash_bin.lua`: 垃圾箱 prefab
- `folkmod_requirement.md`: 开发过程中的需求记录，仅作参考，不代表当前版本全部实现范围

## 已实现功能

### 1. 支持的储物容器扩容

当前只改两个原版容器：

- `treasurechest` 固定扩容为 `300` 格
- `icebox` 固定扩容为 `100` 格

说明：

- 其它原版容器和背包都不改。
- 普通箱子 300 格会使用压缩布局。
- 冰箱 100 格会使用单独布局，避免界面过大。

### 2. 可堆叠物品最大堆叠数提升

所有带 `stackable` 组件的物品在服务端实例化后会调整为默认 `300` 堆叠上限。

### 3. 新增垃圾箱

- 名称：`Trash Bin`
- 配方：`3 boards + 2 goldnugget`
- 容量：固定 `100` 格
- 外观：使用原版木箱动画并整体染成黑色
- 机制：放入的非灰烬物品在默认 `2` 天后自动转成 `ash`

实现方式：

- 计时按槽位保存
- 存档和读档会保留剩余时间
- 新放入的物品重新开始计时

### 4. 切斯特与哈奇强化

- `chester`、`shadowchester`、`snowchester`、`hutch` 固定扩容为 `300` 格
- 设置为无敌
- 添加 `notarget`
- 关闭可被攻击标记

目标是让它们更稳定，不会因为意外吸怪或被打死影响联机体验。

### 5. 队友共享已探索视野

服务端每隔一段时间同步一次当前分片内所有玩家的已探索地图数据：

- 不直接开启全图
- 仅同步已经探索过的区域
- 当前实现以“同分片共享”为主，优先保守稳定

### 6. 草席卷白天可用

给 `bedroll_straw` 补了白天可触发的 `SLEEPIN` 动作入口，尽量沿用原版睡眠动作。

### 7. 冰箱保鲜效率提升

当前对以下容器提升保鲜效果：

- `icebox`
- `saltbox`

默认保鲜倍率为原本的 `2.0x`。

实现方式是直接降低容器的腐坏倍率，不改食物其他属性。

## 暂未完成

当前版本没有实现“容器损坏可修复”与“建筑拆除 1:1 返还原始材料”。

原因：

- 两者都需要接管原版建筑销毁/掉落逻辑
- 为了优先保证联机稳定性，这一版刻意不做高侵入改动
- 当前版本聚焦在仓储、堆叠、垃圾处理、地图共享和轻量体验优化

如果你只想知道当前版本实际做了什么，以本 README 的“功能简单介绍”“已实现功能”“固定规则”为准。

## 固定规则

当前版本不提供可配置项，以下行为全部固定：

- 普通箱子 `300` 格
- 冰箱 `100` 格
- 切斯特和哈奇 `300` 格
- 垃圾箱 `100` 格
- 堆叠上限 `300`
- 垃圾箱转化时间 `2` 天
- 冰箱和盐盒保鲜 `2x`
- 地图共享开启

## 已知限制

1. 这个仓库环境里没有 DST 运行时，当前只能做静态实现和代码级自检，没法直接进游戏联机验证。
2. 没有额外自定义美术资源，垃圾箱和配方图标优先复用原版资源。
3. 普通箱子 300 格、切斯特/哈奇 300 格和冰箱/垃圾箱 100 格都用了压缩布局，不同分辨率下仍可能需要微调。
4. 地图共享当前按同分片周期同步实现，洞穴与地表之间是否需要额外联动，建议你实机测一次。

## 建议测试顺序

1. 先单机开一个新世界，确认模组能正常加载。
2. 测基础仓储：普通箱子、冰箱、切斯特、哈奇。
3. 测垃圾箱：放入普通材料，等待 2 天后是否变灰烬。
4. 测地图共享：两个玩家分别探索不同区域，看是否同步。

## 后续建议

建议下一轮优先做这三件事：

1. 做一次实机联机回归，把地图共享和草席卷白天使用的兼容性确认掉。
2. 按你常用分辨率继续微调普通箱子 300 格和冰箱 100 格界面。
3. 如果后面还想处理“箱子被敲后一地掉落”，建议单独做一个更接近官方思路的简化缓存箱方案。

## 上传到 Steam 创意工坊

如果你想把这个 Mod 上传到 Steam 创意工坊，并设置为仅朋友可见，可以按下面流程做。

### 1. 准备工作

1. 确认你的 Steam 账号拥有《Don't Starve Together》。
2. 在 Steam 里安装 `Don't Starve Mod Tools`。
3. 确认这个 Mod 目录里至少包含：
   - `modinfo.lua`
   - `modmain.lua`
   - `prefabs/`
4. 如果是第一次上传创意工坊内容，先在 Steam 里接受相关订阅协议；否则作品可能只有你自己能看到。

### 2. 安装 Mod Tools

1. 打开 Steam。
2. 进入 `Library`。
3. 切换到 `Tools` 分类。
4. 找到 `Don't Starve Mod Tools` 并安装。

### 3. 上传步骤

1. 启动 `Don't Starve Mod Tools`。
2. 在上传器里选择本地 Mod 目录：
   `/Users/as4k/kyle/gitee.com/xing-dst-folkmod`
3. 填写创意工坊标题、简介和必要标签。
4. 检查封面图、描述和版本信息是否正确。
5. 在上传或更新时，把可见性设置为 `Friends Only`。
6. 确认上传。

### 4. 设置为仅朋友可见

根据 Klei 在 `2021-01-04` 发布的 Mod Tools 更新说明，DST 的上传工具在上传或更新时支持以下可见性选项：

- `Public`
- `Friends Only`
- `Hidden`
- `Unlisted`

你要的效果就是选择：

- `Friends Only`

这样创意工坊页面不会公开给所有人，但你的 Steam 好友可以看到并订阅。

### 5. 上传后建议检查

1. 用你的 Steam 账号打开创意工坊页面，确认作品已经生成。
2. 检查可见性是否确实为 `Friends Only`。
3. 自己先订阅一次，确认能正常下载并在 DST 里启用。
4. 找一个朋友帮你搜索或直接打开链接，确认他们能看到并订阅。

### 6. 常见问题

1. 如果上传成功但别人看不到，先检查是否已经接受 Steam Subscriber Agreement。
2. 如果上传器报错，可以查看 Mod Tools 的日志文件再排查。
3. 如果只是更新现有 Mod，不要删除后重传，直接使用更新功能即可。
