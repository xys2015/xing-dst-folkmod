# 饥荒联机版轻量优化 Mod 开发需求说明

> 说明：本文档保留了开发过程中的需求整理和设计思路，当前仓库实际实现范围请以 `README.md` 为准。

## 1. 文档目的

本文档用于定义 Mod 的功能范围、实现目标、规则边界、兼容性关注点与基础验收标准，供后续开发、联调与测试使用。

本 Mod 面向朋友之间的娱乐联机，目标是降低重复整理、搬运、误损与后勤管理成本，使游戏流程更轻松，但不直接破坏探索、生存与资源经营的核心体验。

## 2. 设计原则

### 2.1 核心方向

1. 优化仓储、整理、运输和基地维护体验。
2. 降低联机中的低价值重复劳动。
3. 减轻因事故、误操作或仇恨机制带来的额外损失。
4. 保留正常探索、收集、建造和生存节奏。

### 2.2 非目标

以下内容不在本 Mod 当前范围内：

1. 不直接开启全图。
2. 不提供玩家无敌。
3. 不提供无限耐久武器或工具。
4. 不直接修改 Boss 战机制。
5. 不改造成完全沙盒化玩法。

## 3. 实现范围总览

本期计划实现以下 7 个功能：

1. 普通箱子固定扩容至 300 格，冰箱固定扩容至 100 格。
2. 所有可堆叠物品堆叠上限提升至 300。
3. 新增 100 格垃圾箱，物品 2 天后自动转化为灰烬。
4. 切斯特与哈奇扩容、无敌，并移除吸引怪物与被攻击特性。
5. 队友共享已探索地图视野，但不开全图。
6. 草席卷支持白天使用。
7. 冰箱内食物保鲜时间提升至原本的 2 倍。

## 4. 功能需求明细

### 4.1 容器扩容

#### 4.1.1 功能目标

仅调整普通箱子与冰箱的容量，以减少常用仓储整理成本，同时避免高范围改动带来的兼容性风险。

#### 4.1.2 规则定义

1. `treasurechest` 固定调整为 300 格。
2. `icebox` 固定调整为 100 格。
3. 其它原版容器不做容量修改。
4. 扩容后容器原有开启方式、交互逻辑、物品存取逻辑保持不变。

#### 4.1.3 边界说明

1. 本功能不覆盖背包、盐盒、鱼箱、龙鳞箱等其它容器。
2. 生物附属容器如切斯特、哈奇由独立功能单独处理。

#### 4.1.5 验收标准

1. 普通箱子可正常显示和使用 300 格容量。
2. 冰箱可正常显示和使用 100 格容量。
3. 存取、堆叠、关闭、打开、重连后数据不丢失。
4. 容器 UI 不出现无法点击、错位或覆盖异常。

### 4.2 堆叠上限提升

#### 4.2.1 功能目标

将所有可堆叠物品的最大堆叠数提升至 300，以提高收纳、运输与集中管理效率。

#### 4.2.2 规则定义

1. 所有原本支持堆叠的物品，最大堆叠数量统一提升至 300。
2. 原本不可堆叠的物品不强制改为可堆叠。
3. 拆分、合并、拾取、掉落等行为应遵循扩展后的堆叠上限。

#### 4.2.3 边界说明

1. 需要注意部分物品可能依赖原始堆叠上限参与特殊逻辑。
2. 与其他修改物品属性的 Mod 并存时，可能存在堆叠规则覆盖冲突。

#### 4.2.4 验收标准

1. 堆叠、拆分、拖拽、交易、存入容器均符合 300 上限。
2. 存档读写与重连后堆叠数量正常。
3. 不出现因超大堆叠导致的物品消失或数量异常。

### 4.3 新增垃圾箱

#### 4.3.1 功能目标

新增一个专用于处理无用物品的功能性容器，减少基地杂物堆积与清理成本。

#### 4.3.2 规则定义

1. 新增建筑“垃圾箱”。
2. 垃圾箱容量为 100 格。
3. 垃圾箱外观应与普通箱子明显区分，主题颜色为黑色。
4. 放入垃圾箱的物品在 2 天后自动转化为灰烬。
5. 转化按物品单独计时或按槽位计时，需要在实现时固定一种规则并保持一致。
6. 制作垃圾箱时，在基础配方上额外增加 2 个金块。

#### 4.3.3 待定实现细节

1. 2 天倒计时是基于单个物品进入时间，还是基于当前槽位最近一次写入时间。
2. 计时期间取出再放入是否重置倒计时。
3. 转化后的灰烬是单份生成还是按原物品数量批量生成。
4. 垃圾箱满格情况下转化为灰烬后的堆叠规则如何处理。

#### 4.3.4 交互要求

1. 玩家应能明确识别该建筑不是普通仓储箱。
2. 建议提供基础提示信息，避免误投高价值物品。
3. 若倒计时机制可视化成本可接受，可考虑增加简单状态提示，但不是首期强制项。

#### 4.3.5 验收标准

1. 垃圾箱可正常制作、放置、打开与存取物品。
2. 计时逻辑稳定，跨天、离线、重连后不失效。
3. 物品到时后稳定转化为灰烬，不出现复制、吞物或异常堆叠。

### 4.4 强化切斯特与哈奇

#### 4.4.1 功能目标

提升切斯特与哈奇作为移动储物单位的稳定性和实用性，减少联机过程中因仇恨或误伤导致的损失。

#### 4.4.2 规则定义

1. 切斯特与哈奇容量统一调整为 300 格。
2. 切斯特与哈奇获得无敌特性。
3. 移除其吸引怪物仇恨的效果。
4. 使其不再成为怪物主动攻击目标。

#### 4.4.3 边界说明

1. 需确认“不会被攻击”是否包含范围伤害、Boss 技能伤害和环境伤害。
2. 若原版存在基于受击触发的特殊行为，需确认是否保留。
3. 需确认切斯特/哈奇形态变化相关逻辑是否仍保持原样。

#### 4.4.4 验收标准

1. 切斯特与哈奇可正常携带、开启和存取 300 格物品。
2. 面对怪物时不会主动吸引仇恨，且不会成为攻击目标。
3. 不因受击判定修改而导致跟随、寻路、召唤或存档异常。

### 4.5 共享探索视野

#### 4.5.1 功能目标

实现队友之间共享已探索区域，提高多人协作效率，减少重复跑图。

#### 4.5.2 规则定义

1. 同一世界中的队友共享已探索地图区域。
2. 仅同步已探索内容，不直接开启未探索区域。
3. 地图同步应在队友探索后合理时间内生效。

#### 4.5.3 边界说明

1. 需要确认共享范围是否覆盖所有玩家，还是仅同一阵营或同一分片下玩家。
2. 需要确认死亡、掉线、重连、新加入玩家时的同步策略。
3. 洞穴与地表是否分别同步，需要在实现时统一定义。

#### 4.5.4 验收标准

1. 两名及以上玩家探索不同区域后，地图视野可正常共享。
2. 新加入或重连玩家可以获得应有的探索同步结果。
3. 不出现全图直接点亮、地图回退或同步卡死。

### 4.6 草席卷白天可用

#### 4.6.1 功能目标

提高时间调整与联机节奏控制的便利性，允许玩家在白天使用草席卷。

#### 4.6.2 规则定义

1. 草席卷可在白天正常使用。
2. 使用后的核心效果保持与原版一致，除时间限制外不额外增强。

#### 4.6.3 边界说明

1. 需确认白天使用时是否会触发与夜间一致的全部逻辑。
2. 需关注多人联机时白天跳时对其他玩家状态和事件的影响。

#### 4.6.4 验收标准

1. 白天可正常执行使用动作。
2. 使用后不会导致卡时间、重复结算或异常中断。
3. 多人场景下时间推进与角色状态结算正常。

### 4.7 冰箱保鲜效率提升

#### 4.7.1 功能目标

适度降低多人联机中的食物保鲜压力，减少频繁烹饪和重复补货。

#### 4.7.2 规则定义

1. 放入冰箱的食物，保鲜时间提升为原本的 2 倍。
2. 仅调整冰箱内保鲜效率，不直接取消腐坏机制。
3. 不修改食物的其他属性，如饱食度、生命值和理智值恢复。

#### 4.7.3 边界说明

1. 需要确认“冰箱”是否包含所有冷藏类容器。
2. 需确认对料理、原材料、药品类或特殊可腐坏物品是否统一生效。

#### 4.7.4 验收标准

1. 食物放入冰箱后腐坏速度符合预期倍率。
2. 取出、转移、重连后保鲜进度计算正常。
3. 不出现保鲜失效、反向加速腐坏或属性错乱。

## 5. 固定规则

当前版本不提供可调配置项，以下行为固定：

1. 普通箱子 300 格。
2. 冰箱 100 格。
3. 堆叠上限 300。
4. 垃圾箱 100 格，2 天转灰烬。
5. 切斯特/哈奇 300 格。
6. 冰箱和盐盒保鲜 2 倍。
7. 地图共享开启。

## 6. 兼容性与风险

### 6.1 高风险功能

以下功能实现时需要优先评估兼容性与存档稳定性：

1. 容器扩容。
2. 堆叠上限提升。
3. 共享探索视野。

### 6.2 主要风险点

1. 与其他修改容器、物品、地图、建筑拆除逻辑的 Mod 发生覆盖冲突。
2. 超大容量容器与超大堆叠可能引发 UI 显示或同步性能问题。
3. 垃圾箱计时逻辑若依赖不稳定事件，可能出现离线失效或重连异常。
4. 地图共享在地表与洞穴、多玩家重连场景下可能存在同步边界问题。

## 7. 建议开发优先级

建议按以下顺序实施：

### 7.1 第一阶段

1. 容器扩容。
2. 堆叠上限提升。
3. 冰箱保鲜效率提升。
4. 草席卷白天可用。

说明：以上功能规则相对直接，适合作为首批稳定功能。

### 7.2 第二阶段

1. 强化切斯特与哈奇。
2. 新增垃圾箱。

说明：以上功能有一定交互和对象范围差异，但风险可控。

### 7.3 第三阶段

1. 共享探索视野。

说明：以上功能涉及同步与多人数据一致性，建议在前两阶段稳定后实现。

## 8. 基础测试清单

### 8.1 单机测试

1. 新档与旧档加载是否正常。
2. 每个功能是否按规则触发。
3. 重连、保存、读档后数据是否正确。

### 8.2 多人测试

1. 玩家同时操作容器、堆叠、切斯特/哈奇时是否同步正常。
2. 地图共享是否在多人环境下稳定。
3. 垃圾箱倒计时是否对所有玩家一致可见。

### 8.3 异常测试

1. 大量物品堆叠与大容量容器下是否出现卡顿或异常。
2. 世界重载、玩家掉线重连后容器和垃圾箱状态是否安全。
3. 与常见功能类 Mod 并存时是否存在显著冲突。
